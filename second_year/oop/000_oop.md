## 💡 What is OOP?

**OOP (Object-Oriented Programming)** is a style of programming where we design and build programs using **classes** and **objects**.

- **Class**: A blueprint for creating objects.
- **Object**: An instance of a class; it has state (data) and behavior (methods).

---

## 🤔 Why Use OOP?

- ✅ Real-world modeling
- ✅ Code reusability
- ✅ Easy maintenance and scalability
- ✅ Better data security

---

## 🔑 OOP Concepts

### 1. 🧱 Class and Object

- **Class**: A template/blueprint.
- **Object**: A real-world entity created from a class.

```java
// Class
public class Car {
    String color;
    void drive() {
        System.out.println("Driving...");
    }
}

// Object
Car myCar = new Car();
myCar.color = "Red";
myCar.drive();
```

---

### 2. 🔒 Encapsulation

Encapsulation is the concept of **hiding internal details** and exposing only whats necessary. This is usually achieved using `private` fields and `public` getters/setters.

```java
public class BankAccount {
    private double balance; // hidden

    public void deposit(double amount) {
        balance += amount;
    }

    public double getBalance() {
        return balance;
    }
}
```

➡️ This protects the data from direct access.

---

### 3. 🧬 Inheritance

Inheritance is one of the core concepts of Object-Oriented Programming (OOP) in Java.  
It allows a class to **reuse** properties and behaviors (like variables and methods) from another class.

In simple terms:  
**A child class can "inherit" everything from its parent class.**

### Here's what it helps with:

- **Code Reuse**: Common features don't have to be written again and again.
- **Structure**: It creates a logical hierarchy among classes.
- **Maintainability**: Changes in the parent class automatically reflect in child classes.
- **Extensibility**: New features can be added easily by extending existing classes.

## ⚠️ Important Points to Remember

- Java **only supports single inheritance** (one class can inherit from one other class).
- Constructors are **not inherited**, but a subclass can call the parent constructor using `super()`.
- Private members of the parent class are **not accessible** directly by the child class.
- All Java classes **automatically inherit** from the base class `Object`.


```java
class Animal {
    void eat() {
        System.out.println("This animal eats food.");
    }
}

class Dog extends Animal {
    void bark() {
        System.out.println("Dog barks.");
    }
}
```

```java
Dog d = new Dog();
d.eat();  // inherited method
d.bark(); // own method
```

### 4. 🧩 Polymorphism

**Polymorphism** means **"many forms."** In Java, it allows methods to behave differently based on the context in which they are used. Polymorphism comes in two types:

- **Compile-time Polymorphism** – achieved through **Method Overloading**
- **Run-time Polymorphism** – achieved through **Method Overriding**

---

#### 🔁 Method Overloading (Compile-time Polymorphism)

Method Overloading allows multiple methods with the **same name** but **different parameter lists**. The method to be executed is determined **at compile time**.

```java
public class Printer {

    public void print(String text) {
        System.out.println("Text: " + text);
    }

    public void print(int number) {
        System.out.println("Number: " + number);
    }

    public void print(String text, int count) {
        for (int i = 0; i < count; i++) {
            System.out.println("Repeat " + (i + 1) + ": " + text);
        }
    }

    public static void main(String[] args) {
        Printer p = new Printer();
        p.print("Hello");
        p.print(42);
        p.print("Hi", 3);
    }
}
```

🧠 **Key Point**: The Java compiler determines which method to call based on the method signature — this is **compile-time polymorphism**.

---

#### 🔄 Method Overriding (Run-time Polymorphism)

Method Overriding occurs when a subclass provides a **specific implementation** of a method that is already defined in its superclass. The method that gets executed is determined **at runtime**, based on the object type.

```java
class Animal {
    void sound() {
        System.out.println("Animal makes a sound");
    }
}

class Cat extends Animal {

    @override
    void sound() {
        System.out.println("Cat meows");
    }
}
```

```java
Animal a = new Cat();
a.sound();  // Output: Cat meows
```

🧠 **Key Point**: Even though the reference is of type `Animal`, the object is of type `Cat`, so the `Cat` version of `sound()` is called — this is **runtime polymorphism**.

---

### 📌 Summary

| Type                    | Technique           | Resolution Time  | Example                  |
|-------------------------|---------------------|------------------|--------------------------|
| Compile-time Polymorphism | Method Overloading | During compilation | `print(String)` vs `print(int)` |
| Run-time Polymorphism     | Method Overriding  | At program execution | `Animal a = new Cat(); a.sound();` |

Polymorphism helps you write **flexible**, **extensible**, and **scalable** code.

### 5. 🎭 Abstraction

**Abstraction** is the OOP principle that helps you focus on **what an object does** instead of **how it does it**.

> "Abstraction hides unnecessary details and shows only the essential information."

This concept simplifies software design by allowing developers to work with higher-level logic without worrying about internal implementation.

---

#### 🔍 Real-Life Example of Abstraction

Imagine driving a car. You press the brake pedal to slow down the vehicle.  
You don't need to understand **how** the braking system works—you just know **what** it does.

This is abstraction: you're interacting with a **simple interface** while the **complex logic is hidden** behind it.

---

### 🧱 How Java Achieves Abstraction

Java supports abstraction through:

- **Abstract Classes** (partial abstraction)
- **Interfaces** (full abstraction)

---

### 🔸 Abstract Class Example

An abstract class can have both **abstract** and **non-abstract** methods. It allows partial abstraction.

```java
abstract class Animal {
    abstract void makeSound();  // abstract method

    void eat() {
        System.out.println("This animal eats food.");
    }
}

class Dog extends Animal {
    void makeSound() {
        System.out.println("Bark");
    }
}
```

---

### 🔹 Interfaces for Full Abstraction

An **interface** is a contract. It tells a class **what to do**, not **how to do it**. 
Interfaces provide **100% abstraction** (before Java 8).

```java
interface Animal {
    void makeSound();  // abstract method
}
```

Classes that implement the interface must provide concrete implementations:

```java
class Dog implements Animal {
    public void makeSound() {
        System.out.println("Bark!");
    }
}

class Cat implements Animal {
    public void makeSound() {
        System.out.println("Meow!");
    }
}
```

```java
public class Main {
    public static void main(String[] args) {
        Animal myAnimal = new Dog();
        myAnimal.makeSound();  // Output: Bark!
    }
}
```

🧠 **Key Point**: The interface `Animal` hides the complexity and just ensures that all animals must be able to `makeSound()`.

---

### ⚖️ Interface vs Abstract Class

| Feature               | Interface                        | Abstract Class                        |
|-----------------------|----------------------------------|----------------------------------------|
| Abstraction Level     | 100% abstraction (Java 7)        | Partial abstraction                    |
| Method Types          | Only abstract (Java 7); can have default & static (Java 8+) | Both abstract and concrete methods     |
| Variables             | `public static final` only       | Instance variables allowed             |
| Constructors          | ❌ Not allowed                    | ✔ Allowed                              |
| Multiple Inheritance  | ✔ Yes (can implement many)       | ❌ Only one superclass allowed          |

---

### ✅ Summary

- Abstraction = **"What an object does"**, not "how."
- Java achieves abstraction using **abstract classes** and **interfaces**.
- Interfaces offer a way to enforce rules without dictating the internal workings.

## 🧪 OOP in Java: Real Examples

```java
public class Student {
    private String name; // encapsulation

    public Student(String name) {
        this.name = name;
    }

    public void introduce() {
        System.out.println("Hi, I’m " + name);
    }
}

class Main {
    public static void main(String[] args) {
        Student s = new Student("Alice"); // object
        s.introduce(); // method
    }
}
```
