# 🎯 Generalization & Regularization

---

# 1. Generalization (Main Goal of ML)

## 📌 Definition
Generalization means:

> A model performs well on **new unseen data**, not just the training data.

A good model learns the **real patterns**, not just memorizing examples.

---

## 📌 Inductive Bias (Semantic Continuity)

### Core Assumption in ML:
> Similar inputs should give similar outputs.

Small input changes → small prediction changes.

Example:
- House price model:
  - 1200 sq ft → \$200k
  - 1210 sq ft → should be close to \$200k

---

## 📌 Generalization Error

Difference between:
- Training performance
- Testing performance

### Formula
```text
Generalization Error = Test Error - Training Error
```

Large gap = usually overfitting.

---

# 2. Underfitting vs Overfitting

| Problem | Underfitting | Overfitting |
|---|---|---|
| Model Complexity | Too Simple | Too Complex |
| Learns Patterns? | No | Memorizes Noise |
| Training Performance | Poor | Excellent |
| Testing Performance | Poor | Poor |
| Bias | High | Low |
| Variance | Low | High |

---

# 📉 Underfitting (High Bias)

## What Happens?
Model is too simple to capture data patterns.

## Signs
- Bad training accuracy
- Bad test accuracy

## Example
Using linear regression for highly nonlinear data.

### Visualization
```text
Simple model → misses important patterns
```

---

# 📈 Overfitting (High Variance)

## What Happens?
Model memorizes training data + noise.

## Signs
- Very high training accuracy
- Poor test accuracy

## Example
High-degree polynomial on small dataset.

### Visualization
```text
Complex curve fits every point exactly
```

## 📌 Bias
Error from overly simple assumptions.

### High Bias
- Model too simple
- Causes underfitting

---

## 📌 Variance
Error from sensitivity to training data changes.

### High Variance
- Model too complex
- Causes overfitting

---

## 📌 Irreducible Error
Noise that cannot be removed.

Examples:
- Measurement errors
- Randomness in data

---

## 🎯 Goal
Find balance:
- Not too simple
- Not too complex

This gives **lowest total error**.

# 4. Regularization

## 📌 Purpose
Regularization reduces overfitting by penalizing large weights.

### Main Idea
```text
Large weights = overly complex model
```

Regularization keeps weights smaller and smoother.

# 5. Lambda (λ) — Penalty Strength

## 📌 Controls Regularization Strength

| λ Value | Effect |
|---|---|
| λ = 0 | No regularization |
| Small λ | Mild regularization |
| Large λ | Strong regularization |
| Very Large λ | Underfitting |

---

## Important Concept
```text
Bigger λ → Smaller Weights
```

---

# 6. Early Stopping

## 📌 Definition
Stop training when validation error starts increasing.

---

## Why?
During training:
- Training error keeps decreasing
- Validation error eventually increases

This means:
```text
Model started memorizing noise
```

---

## Process
1. Train model
2. Monitor validation loss
3. Stop at lowest validation loss
